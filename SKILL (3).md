# Project Planning Skill

## Purpose

Convert broad goals into small, reviewable tasks that Codex can execute safely.

## Workflow

1. Read `AGENTS.md`, `PLANS.md`, `TASKS.md`, `CONTEXT.md`, `DECISIONS.md`.
2. Identify the milestone and business goal.
3. Break work into tasks that each fit one PR.
4. Add dependencies.
5. Add explicit acceptance criteria.
6. Add verification commands where possible.
7. Avoid tasks that mix unrelated concerns.
8. Keep risky actions approval-gated.

## Task format

Each task should include:

- Task ID
- Title
- Status
- Priority
- Type
- Owner
- Dependencies
- Goal
- Scope
- Out of scope
- Acceptance criteria
- Verification
